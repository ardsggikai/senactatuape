GPO-BLOQUEAR-TELA-DO-WINDOWS-APÓS-INATIVIDADE
• User Configuration > Policies > Administrative Templates > Control Panel > Personalization
À direita, serão apresentados os itens de configuração disponíveis para personalização do painel de controle.
Clique duas vezes no item de configuração chamado Habilitar protetor de tela.
Na tela do item de configuração, você precisa selecionar a opção Habilitada.
Clique duas vezes no item de configuração chamado Senha proteja o protetor de tela.
Na tela do item de configuração, você precisa selecionar a opção Habilitada.
Clique duas vezes no item de configuração chamado Tempo limite de salvamento de tela.
Na tela do item de configuração, você precisa selecionar a opção Habilitada.
Defina a quantidade de tempo IDLE que você deseja esperar antes de bloquear a tela do computador.
Em nosso exemplo, estabelecemos um limite de 600 segundos.
Clique duas vezes no item de configuração chamado Economia de tela específica da Força.
Na tela do item de configuração, você precisa definir o seguinte protetor de tela específico:

%windir%\system32\rundll32.exe user32.dll,LockWorkStation

Este comando bloqueará a tela do computador em 10 Minutos vez de mostrar um protetor de tela.

GPO-SCRIPT-DE-CONFIGURAÇÃO-AUTOMÁTICA-DE-PROXY
Instale um servidor IIS para tornar o script de configuração de Proxy públicamente disponível.

Abra o aplicativo Server Manager.

Acesse o menu Gerenciar e clique em Adicionar funções e recursos.

Na tela Funções do servidor, selecione a opção nomeada: IIS do Servidor Web.

Clique no botão Avançar.

Na tela a seguir, clique no botão Adicionar recursos.

Na tela Recursos, clique no botão Avançar.
Na tela de serviço Role, clique no botão Avançar.

Na parte direita da tela, acesse a opção nomeada: MIME Types

Adicione uma nova extensão de arquivo nomeada (.pac) usando o seguinte tipo MIME.

application/x-ns-proxy-autoconfig

Tutorial Windows - Criando um arquivo PAC

Inicie o aplicativo Bloco de Notas e crie um script de configuração automática de Proxy, também conhecido como PAC.

var NETINT="172.16.4.254“;
var NETMASK="255.255.255.0";

function FindProxyForURL(url, host)
{
if (isInNet(host, NETINT, NETMASK))
 	{
  		return "DIRECT";
 	}
else
 	{
  		return "PROXY 172.16.4.254:3128;";
}
}

Este arquivo Proxy PAC especifica que o acesso à rede interna 172.16.4.0/24 deve ser direto.

O acesso à rede interna não deve usar um servidor Proxy.

Para todos os outros acessos, o computador deve usar o servidor Proxy 172.16.4.254 e a porta TCP 3128.

Salve o script de configuração automática do Proxy na raiz do servidor IIS.

C:\inetpub\wwwroot\Proxy.pac

Como exemplo, aqui está nossa configuração.

Você terminou a criação do script automático de configuração do Proxy.

Tutorial GPO - Script automático de configuração do Proxy

No controlador de domínio, abra a ferramenta de gerenciamento de políticas de grupo.

Crie uma nova política de grupo.

User Configuration > Preferences > Control Panel Settings > Internet Settings

Clique com o botão direito do mouse na opção Configurações da Internet e crie um novo item de configuração para o Explorador de Internet (10).
Acesse a guia Conexões e clique no botão de configurações LAN.

Digite a URL no Script de configuração automática do Proxy.
O sistema criará uma preferência de grupo.
Para salvar a configuração da política de grupo, você precisa fechar o editor de Política de Grupo.

Parabéns! Você terminou a criação do GPO.
